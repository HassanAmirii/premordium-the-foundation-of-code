Definition and Purpose


Have you wondered how a computer understand a human readeable code?, Because as you and i both know the only things a computer understand are just 1s and 0s , Take a look at the python code below. 


        
        A = 1
        B = 2

        print(f"i have {A + B} apples")
        

        output: i have 3 apples

The code above is simply easy to understand right, even for a non technical person, But this syntax are pretty much gibberish to the computer, Because it only understand 1s and 0s .


The solution we've got now is to find a way to make the computer understand our gibberish, That is, We need to find a way to turn our code into 1s and 0s (machine language).

So basically whenever you are using a compiler language, your code is proccesed through compiling into an executable file ( 1s and 0s ), and it gives you back the output.

An interpreter language is also another interesting way we can convert our code to binary(machine language) We could also use the waiter and the chef story to simplify this So again you are at a restaurant and approached by a waiter, a pen and a piece of paper was handed to you to write down your order • Biscuit • Bread • Yam Now because you are dealing with an interpreter waiter, your order won’t be processed at once, rather one after the order, That is the waiter will take the first oder on the list which is biscuit here, give it to the waiter to prepare, deliver the order and repeat, until it finish processing the last order, quiet slow compare to the compiler Waiter which processed our order at once.
